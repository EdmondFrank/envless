# envless v0.0.1 — acceptance criteria

Minimum viable, **usable** for one developer migrating one project. Teams + plugins + skill come in v0.0.2+.

## Must work end-to-end

```bash
# 1. Identity
envless init                     # creates .envless/, age identity, .sops.yaml (current user only)

# 2. Store a secret
echo "sk-test-xyz" | envless set OPENAI_API_KEY

# 3. Read it back (guarded)
envless get OPENAI_API_KEY --confirm

# 4. Inject into a process — THE STAR COMMAND
envless exec -- env | grep OPENAI_API_KEY
# expected: OPENAI_API_KEY=sk-test-xyz

# 5. Multi-env
echo "sk-prod-xyz" | envless set OPENAI_API_KEY --env=prod
envless exec --env=prod -- env | grep OPENAI_API_KEY

# 6. Migration
envless migrate .env             # encrypts .env → secrets/dev.yaml.enc, adds .env to .gitignore

# 7. List
envless list                     # show keys (not values) per env
```

## Out of scope for v0.0.1

- Plugins (deferred to v0.1)
- Teams (deferred to v0.1)
- Skill / npx install (deferred to v0.1)
- GH Actions auto-patch (deferred to v0.1)
- Rotation adapters (deferred to v0.2)
- Panic mode (deferred to v0.2)
- Inbox / notice (deferred to v0.2)

## Risky parts (TDD priority)

| Priority | Module | Risk |
|---|---|---|
| 1 | `zig/src/execenv` | wrong env array = wrong process. Highest impact bug. |
| 2 | `zig/src/envparse` | `.env` quirks (quotes, multiline, $VAR interp) — silent data corruption |
| 3 | `zig/src/sops` | wraps `sops` binary; roundtrip must be lossless |
| 4 | `zig/src/store` | secrets/*.env.enc layout; future-compatibility |
| 5 | `zig/src/cli/exec` | glue: where bugs surface to user |
| 6 | `zig/src/cli/set` / `cli/get` | stdin handling, no plaintext echo |
| 7 | `zig/src/cli/migrate` | destructive (touches .gitignore); test idempotency |

## Test policy

- TDD: red → green → refactor per slice.
- Inline `test "..."` blocks per Zig module, table-driven where it pays off.
- E2E in `zig/src/e2e.zig` (Zig test harness shelling out to the built
  binary), uses real `sops`+`age` (skipped via `error.SkipZigTest` when
  missing).
- Caveman output: one line per test, transitions only in watch mode.
- Suite wall < 10s for v0.0.1.

## Done when

- [ ] All 7 commands in "Must work end-to-end" pass automated test.
- [ ] `zig build test` green (unit tests).
- [ ] `zig build e2e` green (end-to-end).
- [ ] `zig build` produces `zig-out/bin/envless` that runs the manual
      test plan above.
- [ ] README updated with quickstart.
- [ ] Tagged v0.0.1.
